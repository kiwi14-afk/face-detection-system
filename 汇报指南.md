# 人脸检测系统 · 项目汇报指南

## 项目概述

基于 YuNet 的人脸检测系统，包含图像预处理管线、PyTorch 模型训练、WIDER FACE 评测和 Web 交互应用。

**技术栈：** Python · PyTorch · OpenCV · FastAPI · HTML/CSS/JS  
**模型：** YuNet（~85K 参数，anchor-free，多尺度检测 stride 8/16/32）  
**数据集：** WIDER FACE（训练 12,880 张 + 验证 3,226 张）

---

## 成员A：数据准备 + 图像预处理

### 负责文件

| 文件 | 行数 | 功能 |
|------|------|------|
| `preprocessing/process_all.py` | 100 | 预处理核心 + 批量处理 |

### 工作内容

#### 1. 数据集准备
- 下载 WIDER FACE 数据集（训练集 12,880 张 + 验证集 3,226 张）
- 整理目录结构，确保标注文件与图片一一对应
- 数据存放于 `preprocessing/3226_all/`，按 WIDER FACE 原有子目录组织（59 个类别文件夹）

#### 2. 图像预处理管线
- **CLAHE**（对比度受限自适应直方图均衡化）：在 LAB 色彩空间的 L 通道上操作，补偿光照不均匀，增强阴影区域细节
- **双边滤波**：在保边的前提下去除噪声，参数 d=5, σ_color=25, σ_space=25
- **USM 锐化**（反锐化掩膜）：对滤波后的图像进行适度锐化，增强人脸五官边缘，amount=1.5, radius=1.5
- 三级管线串联：CLAHE → 双边滤波 → USM，在去噪与保留高频特征之间取得平衡

#### 3. 批量处理
- `batch_process_full_dataset()` 递归遍历输入目录所有 `.jpg`
- 保持原始子目录结构输出
- 每 500 张打印进度，支持断点续传思路

### 汇报要点

> **开场（1 分钟）：** 我负责的是数据准备和图像预处理。我们使用的是 WIDER FACE 数据集，包含一万两千多张训练图片和三千多张验证图片。为了提高模型在低光照、模糊等困难场景下的检测效果，我设计了一套三级预处理管线。

> **技术细节（2 分钟）：**
> 1. 先用 CLAHE 做光照补偿——把图像转到 LAB 空间，只在亮度通道上做自适应直方图均衡化，避免色彩失真
> 2. 然后用双边滤波去噪——和普通高斯滤波不同，双边滤波考虑了像素的空间距离和颜色差异，能在去噪的同时保住人脸边缘
> 3. 最后用 USM 锐化——高斯模糊后与原图做差值再叠加回去，适度增强五官轮廓
>
> 预处理后肉眼可见对比度提升、噪点减少。最终对整个验证集做了全量批处理，保持原始目录结构输出。

> **关键数字：** 3 级管线 / 3,226 张图片批处理 / CLAHE clip_limit=3.0

## 成员B：模型训练系统

### 负责文件

| 文件 | 行数 | 功能 |
|------|------|------|
| `training/yunet_torch/model.py` | ~200 | YuNet 网络架构 |
| `training/yunet_torch/loss.py` | ~150 | 多任务损失函数 |
| `training/yunet_torch/dataset.py` | ~150 | 数据加载器 |
| `training/yunet_torch/train.py` | ~400 | 训练循环 + ONNX 导出 |
| `training/yunet_torch/__init__.py` | 3 | 包定义 |
| `training/launch_full.py` | 37 | 训练启动配置 |
| `training/outputs/.../final_model.pth` | — | 最终模型权重（~700KB） |

### 工作内容

#### 1. YuNet 网络架构（model.py）
- **YuNetBackbone**：轻量级卷积骨干网络，通过步长 2 的卷积逐级下采样
- **TFPN**（Tiny Feature Pyramid Network）：将不同深度的特征图融合，输出 3 个尺度（stride 8/16/32）
- **YuNetHead**：每个尺度上同时预测分类、边界框回归和 Objectness，实现 anchor-free 检测
- 总参数量约 85K，适合端侧部署

#### 2. 损失函数（loss.py）
- **分类损失**：交叉熵，区分人脸/背景
- **回归损失**：EIoU Loss，同时优化 bbox 中心点、宽高和长宽比
- **Objectness 损失**：衡量"该位置是否包含人脸"的置信度
- 三部分加权求和，处理正负样本不平衡

#### 3. 数据加载（dataset.py）
- 解析 WIDER FACE 标注格式（图片路径 + 人脸数 + bbox 列表）
- 支持数据增强（随机翻转、尺度扰动等）
- 对每张图 resize 到 640×640 并做 padding 保持宽高比

#### 4. 训练循环（train.py）
- 完整训练/验证流程，TensorBoard 日志记录
- 学习率 warmup + cosine 衰减策略
- 定期 checkpoint 保存（每 20 epoch）
- 训练结束后自动导出 ONNX 模型（opset 11）

#### 5. 训练配置（launch_full.py）
- 100 epoch, batch_size=16, lr=0.01
- 训练集 10,304 张，验证集 2,576 张（从 WIDER FACE 训练集 80/20 划分）
- 输入分辨率 640×640，无关键点输出（use_kps=False）

### 汇报要点

> **开场（1 分钟）：** 我负责的是核心的模型训练系统。YuNet 是一个轻量级人脸检测模型，我在 PyTorch 下从零实现了完整的训练管线，包括网络架构、损失函数、数据加载和训练循环。

> **技术细节（2-3 分钟）：**
> 1. **架构**：YuNet 采用 anchor-free 设计，不需要预设锚框。骨干网络逐级下采样后经过 TFPN 特征金字塔融合，输出 3 个尺度（8/16/32 倍下采样），每个位置直接预测分类分数、bbox 偏移和 objectness。总参数只有 8.5 万，非常轻量。
> 2. **损失函数**：三个任务联合训练——分类用交叉熵区分人脸/背景，回归用 EIoU Loss 同时优化位置和形状，Objectness 衡量该位置有脸的概率。最终检测置信度 = 分类分 × Objectness。
> 3. **训练**：在 WIDER FACE 一万张预处理图像上训练 100 轮，batch 16，学习率 0.01 配合 warmup 和余弦衰减。训练完自动导出 ONNX 方便部署。

> **关键数字：** ~85K 参数 / 100 epoch / 10,304 训练图片 / 640×640 输入 / 3 个检测尺度

## 成员C：Web 交互应用 + 模型评测

### 负责文件

| 文件 | 行数 | 功能 |
|------|------|------|
| `webapp/server.py` | 173 | FastAPI 后端服务 |
| `webapp/static/index.html` | 320 | 前端交互界面 |
| `training/eval_new.py` | 143 | WIDER FACE 评测脚本 |

### 工作内容

#### 1. 模型评测（eval_new.py）
- 在 WIDER FACE 验证集（3,226 张）上全量评测
- 单尺度 640×640 推理，与 Web 服务共用 Decode+NMS 逻辑
- 多阈值（0.03~0.10）输出 Precision / Recall / F1
- 基于 IoU=0.5 的贪心匹配策略判定正误检

#### 2. 后端 API（server.py）
- **框架**：FastAPI + uvicorn，端口 7860
- **模型加载**：启动时加载成员B训练的 `final_model.pth`，CUDA/CPU 自适应
- **预处理集成**：调用成员A的 `FaceImagePreprocessor`，支持一键开关对比
- **推理流程**：原图 + 预处理图分别检测 → 基于 IoU 匹配配对 → 输出对比分析
- **`POST /api/detect` 接口**：
  - 入参：图片（multipart）、预处理开关、置信度阈值
  - 出参：原始检测结果、预处理检测结果、置信度对比（升/降/不变/消失/新增）、预处理图像 base64

#### 3. 前端界面（index.html）
- **布局**：左侧控制面板（上传、开关、滑块），右侧双画布并排（原始图 vs 预处理图）
- **画布**：Canvas 绘制检测框，颜色编码置信度——青色 ≥0.8、黄色 ≥0.6、红色 <0.6
- **置信度可视化**：双栏柱状图，每对检测结果直观对比原始/预处理置信度
  - ↑ 绿色（提升）、↓ 红色（下降）、≈ 灰色（不变）、✕ 消失、＋ 新增
- **统计面板**：人脸计数（原始/预处理）、单次耗时、预处理状态
- **指标表格**：WIDER FACE 验证集多阈值 F1/Precision/Recall 数据展示
- **主题**：暗色/亮色一键切换，纯 HTML/CSS/JS 无框架依赖

### 汇报要点

> **开场（1 分钟）：** 我负责的是评测和 Web 应用。评测方面在 WIDER FACE 验证集上对模型做了多阈值精度测试；Web 方面搭建了一个完整的人脸检测交互平台，支持实时检测和预处理效果对比。

> **评测部分（1 分钟）：**
> 在 3,226 张验证图上跑了多阈值评测，用的是和 Web 服务同一套推理逻辑。输出 Precision / Recall / F1，最好阈值下 F1 约 0.32。评测脚本直接加载成员B训练好的权重即可复现。

> **Web 部分（2 分钟）：**
> 后端用 FastAPI，加载训练好的 PyTorch 模型做推理。核心功能是对比——同一张图分别在原始状态和经过成员A的预处理管线后各检测一次，用 IoU 把两次结果配对，算出每张人脸的置信度变化（上升还是下降），以双栏柱状图直观展示。预处理图也通过 base64 编码传到前端画布上，可以和原图并排看。
>
> 前端是纯原生 HTML/CSS/JS，没用到任何框架。Canvas 画检测框、动态柱状图、拖拽上传、阈值滑块都是手写的。

> **关键数字：** 3,226 张评测 / 多阈值 P/R/F1 / 1 个 API 端点 / 320 行前端代码

## 汇报流程建议

### 顺序（总时长 8-10 分钟）

```
成员A（2 分钟）→ 成员B（3 分钟）→ 成员C（3 分钟）→ 现场演示（2 分钟）
```

### 演示准备

1. 启动服务：`cd webapp && python server.py`
2. 打开浏览器访问 `http://127.0.0.1:7860`
3. 准备 2-3 张测试图片（建议包含：正常光照人像、暗光人像、多人合影）
4. 现场演示流程：
   - 上传图片 → 观察检测结果
   - 切换预处理开关 → 对比置信度变化
   - 调节阈值滑块 → 观察人脸数量变化
   - 展示暗色/亮色主题

### 可能被问到的问题

| 问题 | 准备回答 |
|------|---------|
| 为什么选 YuNet？ | 轻量（85K 参数）、anchor-free 设计简洁、适合学习和部署 |
| 模型效果怎么样？ | F1 约 0.32@阈值 0.08，小模型在 WIDER FACE 上属于正常水平 |
| 预处理真的有用吗？ | 低光照场景下 CLAHE 提升明显，Web 端可直观看到置信度 ↑ |
| 能实时跑吗？ | CPU 下单张约 45ms（约 22 FPS），CUDA 下可达实时 |
| 为什么不用 ONNX？ | 训练时已导出 ONNX，但 Web 端直接用 PyTorch 方便调试 |
| 你们怎么分工的？ | A 做数据和预处理，B 做模型训练，C 做评测和 Web |
